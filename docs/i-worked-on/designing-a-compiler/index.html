Written in OCaml, my compiler generates X86-64 from my progrmming language, with C as the run-time. My language features tests as first-class citizens, that can be compiled away for release.

Garbage Collection

A stop-the-world garbage collector, by applying graph theory to implement Cheney's semispace collector, implemented with C.

A-normal Form (ANF)

Data Structures

There is a compiler phase that turns the abstract syntax tree to ANF, an intermediary representation that makes every intermediate computation named. This makes it easier to compile down to Assembly.

Graph Colouring Register Allocation

AlgorithmsData Structures

Accessing a value in a register is faster than accessing it on the stack, so we want to keep values in registers for as long as possible. To do this, we track the liveness of each value — the range of the program over which it is needed — and use Chaitin's graph colouring algorithm to assign values to registers optimally

Built-in Testing Functionality

Normally, to run tests in a programming language, one has to use a library. This makes test organization awkward, and the library would have to use reflection magic to test for errors. In this programming language, tests are first-class citizens and can be woven into the code, and the code can be executed without the tests. The C run-time accumulates the test information and prints them out at the end.
